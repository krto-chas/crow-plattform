# CAS-919 – Federation Metadata Architecture

## Purpose

Defines the normative architecture for metadata management across CAF-governed federated environments.

## Scope

Applies to business, technical, operational and security metadata exchanged or managed within the federation.

## Metadata Principles

- Metadata SHALL have an identified owner.
- Metadata SHALL be versioned.
- Metadata SHALL be traceable to governed assets.
- Metadata SHALL support interoperability.

## Metadata Model

Metadata SHALL describe identity, ownership, classification, lifecycle state, provenance and applicable governance where relevant.

## Lifecycle

Created → Approved → Published → Maintained → Deprecated → Archived.

## Governance

Metadata changes SHALL follow documented governance and be auditable.

## Conformance

Implementations SHALL demonstrate governed metadata, lifecycle management, ownership and traceability.
