# CAS-901 – Architecture Classification

## Purpose

Define the normative classification model for architecture artefacts managed within Crow Architecture Framework (CAF).

## Scope

This standard applies to all governed artefacts published within CAF repositories.

## Classification Principles

Architecture artefacts SHALL be classified consistently according to their architectural role, lifecycle and governance requirements.

## Classification Categories

- Foundation
- Standard
- Architecture Decision Record (ADR)
- Template
- Reference Architecture
- Compliance Artefact
- Release Artefact
- Supporting Documentation

## Classification Rules

Each artefact SHALL belong to one primary classification category. Secondary tags MAY be used to improve discovery but SHALL NOT replace the primary classification.

## Metadata Requirements

Every classified artefact SHALL include its identifier, title, owner, version, lifecycle status and classification.

## Conformance

A CAF implementation SHALL classify all governed artefacts using this model and maintain the classification throughout the artefact lifecycle.

## Summary

Architecture Classification establishes a consistent structure for organizing, governing and locating architecture artefacts.
