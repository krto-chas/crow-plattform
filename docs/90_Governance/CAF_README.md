# Crow Architecture Framework (CAF)

CAF is the small set of governance documents around the Crow repositories. The architecture itself lives in the repository (`docs/`) and in code — CAF points, it never restates (CAF-002 §7).

| Document | Purpose |
|---|---|
| CAF-001 Glossary | normative terminology, anchored in code |
| CAF-002 Architecture Principles | the seven principles + repository conventions |
| CAF-CMP-001 Conformance Checklist | the actual gates: CI, module, release |
| CAF-TPL-002 ADR Template | the one ADR form (repository series only) |
| CAF-TPL-003 Architecture Review Checklist | pre-tag review, auto + human rows |
| CAF-TPL-004 Standard Change Request | external change requests |
| CAF-REL-001 Release Manifest | the B*_VERIFICATION.json format |
| CAF-REL-002 Release Notes | CHANGELOG structure and rules |
| CAS-900 Architecture Overview | index over repository architecture docs |
| CAF-FUT-001 Federation Roadmap | non-normative direction note |
| MIGRATION_FROM_RC1 | mapping from the original 109-document corpus |

Growth rule: CAF stays at ~10 active documents. If it needs more, the content belongs in `docs/`.
