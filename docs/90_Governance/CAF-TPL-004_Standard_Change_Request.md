# CAF-TPL-004 Standard Change Request

For change requests from parties outside the repository (module partners, pilot customers). Internal changes go straight to an ADR or issue.

```markdown
# CR-<yyyy>-<nnn>: <Title>

**Requested by:** <organization / person>
**Date:** <date>
**Affects:** <packages / contracts / documents>

## Requested change
What should change, stated concretely.

## Motivation and evidence
Why — with the document, defect or measurement that motivates it.

## Compatibility assessment
Breaking / additive / internal. If breaking: which version line.

## Decision
Accepted (→ ADR-nnn or CHANGELOG ref) | Rejected (reason) | Deferred (condition)
```
